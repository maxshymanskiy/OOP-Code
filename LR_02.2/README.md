У класі IntegerSet змінні size та capacity мають різне призначення. Ось детальне пояснення:

size_t size;
Призначення: Зберігає поточну кількість елементів у множині.

Приклад: Якщо множина містить елементи {1, 2, 3}, то size = 3.

Використання:

Вказує, скільки елементів фактично зберігається в масиві elements.

Використовується для ітерації по елементах множини, додавання нових елементів тощо.

size_t capacity;
Призначення: Зберігає загальну ємність масиву elements, тобто максимальну кількість елементів, яку може вмістити масив на даний момент.

Приклад: Якщо capacity = 10, то масив elements може вмістити до 10 елементів, навіть якщо фактично зберігається менше (наприклад, size = 3).

Використання:

Використовується для керування пам'яттю. Якщо size досягає capacity, потрібно збільшити capacity (виділити більше пам'яті).

Дозволяє уникнути постійного перевиділення пам'яті при додаванні кожного нового елемента.

Різниця між size та capacity
Параметр	size	capacity
Значення	Поточна кількість елементів	Максимальна кількість елементів, яку може вмістити масив
Приклад	Якщо множина {1, 2, 3}, то size = 3	Якщо capacity = 10, то масив може вмістити до 10 елементів
Зміна	Збільшується при додаванні елемента	Збільшується при необхідності (коли size досягає capacity)
Використання	Для роботи з елементами множини	Для керування пам'яттю (виділення/звільнення)
Приклад роботи з size та capacity
Початковий стан:

Масив elements порожній.

size = 0 (немає елементів).

capacity = 0 (пам'ять не виділена).

Додавання першого елемента:

Виділяється пам'ять (наприклад, capacity = 1).

Додається елемент 1.

size = 1, capacity = 1.

Додавання другого елемента:

Оскільки size == capacity, потрібно збільшити capacity (наприклад, удвічі: capacity = 2).

Додається елемент 2.

size = 2, capacity = 2.

Додавання третього елемента:

Знову size == capacity, тому capacity збільшується (наприклад, до 4).

Додається елемент 3.

size = 3, capacity = 4.

Чому важливо розрізняти size та capacity?
Ефективність:
Збільшення capacity заздалегідь (наприклад, удвічі) дозволяє уникнути постійного перевиділення пам'яті, що є дорогою операцією.

Контроль пам'яті:
capacity дозволяє знати, скільки пам'яті виділено, а size — скільки фактично використовується.

Уникнення помилок:
Якщо size перевищує capacity, це призведе до виходу за межі масиву та невизначеної поведінки програми.

Приклад коду:

void IntegerSet::addElement(int elem) {
    if (size >= capacity) { // Якщо масив заповнений
        resize(capacity == 0 ? 1 : capacity * 2); // Збільшуємо ємність
    }
    elements[size++] = elem; // Додаємо елемент і збільшуємо size
}
Висновок:
size — це кількість елементів, які фактично зберігаються в множині.

capacity — це загальна ємність масиву, яка може бути більшою за size.

Розрізнення цих змінних дозволяє ефективно керувати пам'яттю та уникнути помилок.
